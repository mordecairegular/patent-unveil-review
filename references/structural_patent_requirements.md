# 结构类专利撰写规范参考

> **使用范围**：仅在 `prompts/intake.md` Q1.5 B 路线（结构有限辅助）时引用。本文件提供 CNIPA 附图规范、结构权利要求格式、部件编号和文字清洁参考，不用于结构类方案的专利点挖掘或创造性评估。本技能处理结构类案件时最高输出 WARN。

本文件汇总 CNIPA（中国国家知识产权局）对机械/结构类发明专利和实用新型专利的附图、权利要求和说明书撰写规范要求。供 `disclosure_builder.md` 和 `disclosure_self_check.md` 在结构有限辅助通道下引用。

来源：《专利法》第26条、《专利法实施细则》第18/20条、《专利审查指南》第一部分第一章、WIPO 专利撰写指南。

## 一、附图规范（CNIPA 硬性要求）

### 1.1 线条与格式

- 附图**必须**使用黑色墨水绘制的线条图（或计算机绘图输出的黑白线图）
- **不得**使用：照片、灰度阴影、渐变填充、彩色、半色调（halftone）
- 线条应清晰、均匀，不同粗细可用于区分不同部件或截面
- 剖面线用等距平行斜线表示，不同材料的剖面线方向或间距应有所区别

### 1.2 部件编号（附图标记）

- **必须**使用阿拉伯数字编号（1, 2, 3, ...）
- 编号用引出线（指引线）指向对应部件，引出线不得交叉
- **同一部件在所有附图中编号必须一致**
- 编号不应直接标注在部件上，而应通过引出线从部件引出到图框空白处
- 附图中**不得**包含文字说明（「附图标记与说明书文字互斥」原则），仅允许数字编号和必要的尺寸标注

### 1.3 附图说明书中的引用格式

- 说明书中引用附图标记时，格式为「部件名称 编号」
- **不加括号**：「插针导体 3」而非「插针导体（3）」或「插针导体(3)」
- 审查指南明确规定：附图标记放在技术名称后面

### 1.4 附图类型与适用场景

| 附图类型 | 适用场景 | 说明 |
|---------|---------|------|
| 整体剖面图 | 展示内部结构和部件位置关系 | 最常用，可标注剖切面 |
| 局部放大图 | 展示关键部件的细节结构 | 用圆圈或方框标出放大区域 |
| 分解图（爆炸图） | 展示装配关系 | 各部件沿装配方向分离 |
| 状态图（多张） | 展示不同工作状态下的结构变化 | 每个关键状态一张，编号连续 |
| 原理图/示意图 | 展示动作顺序、力学关系 | 可简化结构细节 |
| 剖视图 | 展示截面形状 | 标注剖切位置 |

### 1.5 常见错误

- ❌ 使用 AI 生成的像素图（灰度、渐变、不精确的结构关系）
- ❌ 使用 3D 渲染图（有阴影和光照效果）
- ❌ 在附图上标注文字说明（只允许数字编号）
- ❌ 不同附图中同一部件使用不同编号
- ❌ 术语错误（如「男端」应为「公端」或「公头」）

## 二、结构类权利要求撰写要则

### 2.1 独立权利要求（两段式）

结构类发明专利的独立权利要求通常采用"两段式"写法：

```
一种[产品名称]，包括[前序部分：已知结构要素]，
其特征在于：[特征部分：创新的结构要素及其关系]
```

**前序部分**：包含与最接近现有技术共有的必要技术特征
**特征部分**：包含区别于最接近现有技术的必要技术特征

### 2.2 结构特征限定优先级

撰写权利要求时，优先使用以下类型的限定（按优先级排序）：

1. **形状特征**：部件的几何形状（环形、弧形、锥形等）
2. **构造特征**：部件的内部构造（中空、实心、分层等）
3. **连接关系**：部件之间的连接方式（固定连接、滑动配合、铰接等）
4. **位置关系**：部件之间的空间位置（同轴、平行、垂直、包围等）
5. **材料特征**：必要时限定材料类型
6. **尺寸/比例关系**：必要时限定关键尺寸的相对关系

### 2.3 应避免的限定方式

- **纯功能性限定**：「用于密封的部件」→ 应改为「设置于…的环形密封圈」
- **方法步骤混入结构独权**：结构独权中不应出现操作动作
- **不确定用语**：「大约」「左右」「优选」等在独权中应避免
- **过度限定**：将可选特征写入独权会不必要地缩窄保护范围

### 2.4 从属权利要求策略

从属权利要求应形成**金字塔式**保护体系：

```
独权1：核心结构（保护范围最宽）
  ├─ 从属2：具体形状细化
  ├─ 从属3：材料限定
  ├─ 从属4：尺寸/比例关系
  ├─ 从属5：附加结构部件
  ├─ 从属6：替代实施方式
  └─ 从属7-N：更具体的结构变型
```

## 三、说明书撰写要则

### 3.1 结构类说明书的特殊要求

- **结合附图说明**：具体实施方式部分必须结合附图进行说明
- **部件首次出现**时给出编号和名称（如「插针导体 3」），后文可省略编号但名称需一致
- **状态/工位描述**：按动作顺序或状态变化顺序组织，每个状态对应具体附图
- **技术闭环**：从初始状态到终态的完整技术路径，不得在中间断裂

### 3.2 常见结构类撰写问题

| 问题 | 表现 | 改进方法 |
|------|------|---------|
| 主实施例发散 | 同一部件列举5种以上替代形态 | 主方案固定一种，替代放实施例 |
| 功能性描述代替结构描述 | 「用于保护的件」无形状/连接/位置 | 补充具体结构特征 |
| 技术闭环断裂 | 状态描述到中间步骤就停止 | 补充终态描述和附图 |
| 报告式语言 | 「经过分析」「研究发现」 | 删除，直接写技术方案 |
| 附图标记不一致 | 同一部件在不同段落编号不同 | 全文统一核查 |
| 参数无来源 | 具体数值无实验/仿真依据 | 改为设计约束关系 |

### 3.3 WIPO 10步撰写法（适用于结构类）

1. **先画附图**：确定需要多少张图、每张图展示什么状态/视角
2. **写独立权利要求**：基于附图确定核心创新结构
3. **写从属权利要求**：逐层细化结构特征
4. **写具体实施方式**：结合附图逐状态描述
5. **写发明内容/技术方案概述**：从具体实施方式中抽象
6. **写背景技术**：对比现有技术的结构缺陷
7. **写技术问题**：对应背景技术的不足
8. **写有益效果/优点**
9. **写摘要**
10. **全文交叉核查**

### 3.4 "基础实施例法"

先写一个**中位概念的实施例**（不过于具体也不过于抽象），作为独立权利要求对应的实施方式。然后：
- 向具体方向写 2-3 个变型实施例（具体材料、具体尺寸、具体应用场景）
- 向抽象方向检查独立权利要求是否还能进一步上位

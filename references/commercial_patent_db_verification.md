# 商业专利库内容核验与来源状态

适用场景：用户已授权并打开壹专利、高数图等商业专利库，或要求用商业库减少人工查新/复核工作。

## 核心原则

商业专利库不是“不能用”，也不是“可直接公开引用”。Agent 应主动完成可合规自动化的检索、详情读取、对比和证据落盘，减少用户和代理人的机械核验负担；但商业库学校 VPN/session URL 不得写作公开稳定引用，商业库内容核验也不得表述为代理人的正式法律检索意见。

## 三类状态

```yaml
commercial_db_discovered:
  meaning: 在壹专利、高数图等商业库结果列表中发现候选文献
  can_support_internal_search: true
  can_support_content_comparison: false
  can_be_public_citation: false

commercial_db_content_checked:
  meaning: 已在商业库详情页读取摘要、著录项/IPC、权利要求或说明书要点
  can_support_internal_search: true
  can_support_content_comparison: true
  can_be_public_citation: false

public_source_verified:
  meaning: 已用 CNIPA PSS、中国专利公布公告、Google Patents、Espacenet、WIPO 等公开稳定来源核验
  can_support_internal_search: true
  can_support_content_comparison: true
  can_be_public_citation: true
```

## Agent 必做

1. 用户已打开授权页面时，按需使用该页面低频检索，不要笼统回答“留给代理人/用户复核”。
2. 对交底书 1.1.2 表格中的中国公开文献，优先按公开号主干检索，例如 `CN115360706`，不要首选带 A/B kind code 的完整号。
3. 每条候选至少记录：平台、检索式、结果行、公开号、题名、申请人、公开日、法律状态显示、IPC、摘要要点、权利要求读取状态、相关性等级、URL 稳定性。
4. 查询结果必须落盘到案件证据包，不能只在对话中声称“查到了”。
5. 证据状态变化后，回写案件证据包、交接说明、交付检查记录，并把可用于区别论证的技术事实回写到交底书 1.1.2、1.1.3。
6. 回写正文后重新生成 DOCX，并执行公式/图示 QA 与正式正文清洁 QA。

## 壹专利稳定流程

1. 使用用户已打开的学校远程访问壹专利页面，不自行猜测直连入口。
2. 每次检索前回到壹专利首页或干净搜索页。
3. 搜索公开号主干，例如 `CN109492824`。
4. 从结果列表点击目标公开号进入详情页。
5. 读取摘要、主著录项/IPC、权利要求页签。
6. 保存 JSON/Markdown 证据，必要时保存截图索引。
7. 遇到验证码、风控、登录过期或权限页，停止并记录，不绕过。

避免在结果页连续改写检索式；该页面状态容易混入旧条件或形成异常检索式。

## 正文与交接说明分离

商业库内容核验完成后，**不得**把核验过程、平台名称、会话 URL、来源状态解释或“正式公开引用仍建议……”这类交接提示写入正式交底书正文。交底书正文只保留对区别论证有用的技术事实：相近技术解决什么问题、采用什么方案、与本发明有哪些相同点和差异。

禁止写入正式交底书正文的典型句式：

```text
下表中国公开专利文献已通过壹专利商业库完成内容核验……
商业库会话 URL 仅作为内部查新证据……
正式申请文件中的公开引用仍建议以 CNIPA PSS……闭环。
壹专利已完成商业库内容核验；正式公开引用仍建议……
详细检索记录、阳性对照和来源状态已另行保存在案件证据包中……
```

正式交底书 1.1.2 推荐写法：

```text
现有公开技术中，与本发明较为接近的方向包括分时价格建模、源荷储协同调度、绿电交易结算和项目收益测算等。下表列举的公开文献均涉及上述一个或多个方向，但其技术目标、处理对象或计算链条与本发明仍存在差异。
```

推荐表格列：

```text
公开文献 | 技术方向 | 与本发明的接近点 | 与本发明的主要区别
```

下列内容写入 `prior_art_dossier.md`、`query_log.md`、`positive_controls.md`、`unverified_sources.md`、`交底书交接说明_*.md` 或 `交底书交付检查记录.md`：

- 使用了哪个商业库、检索式、命中数、截图编号。
- `commercial_db_discovered` / `commercial_db_content_checked` / `public_source_verified` 等来源状态。
- 哪些公开源尚需补齐，哪些 URL 只作内部定位。
- 代理人或用户需要关注的复核边界。

## 可继续自动化但不能伪造

- CNIPA PSS、中国专利公布公告、Google Patents、Espacenet、WIPO 等公开来源闭环可以由 Agent 继续尝试补齐。
- 若官方平台可访问，记录稳定公开链接或可复核来源路径。
- 若官方平台需要登录、验证码、访问受限或结果不可稳定复现，记录阻断原因，不得伪造 `public_source_verified`。

## 仍需代理人/用户确认

- 新颖性、创造性、侵权或 FTO 的正式法律意见。
- 正式申请文件是否列入某篇现有技术，以及列入程度。
- 权利要求保护范围最终取舍。
- 无法由 Agent 合规访问的平台登录、验证码或权限异常处理。
- 最终提交前的 Word 页面级目视复核。

# 边界与输入录入（Step 1）

## 用途

在开始专利挖掘与交底书撰写前，用**少量问题**收敛案件边界。信息不全时可跳过，由 Agent 后续推断并**注明假设**。

## 可选开场

```
为便于挖掘专利点与组织交底书，先确认几项边界；可跳过，将按已有材料推断。
```

## 问题序列

### Q1：技术主题或产品模块

```
请用一句话描述本方案所属技术主题或产品模块，例如「多模态检索排序」「工业质检缺陷分级」「高海拔光伏连接器」「旋转机械密封结构」。
```

### Q1.5：专利技术领域（路由关键问题）

```
本案属于哪种技术领域？
A. 软件/算法/方法类（数据处理、AI、通信协议、控制策略等）
B. 机械/结构/装置类（零部件、连接器、机构、壳体、密封件等）
C. 两者兼有（方法+装置联合保护）
D. 暂不确定
```

**路由规则**：

- **A → 软件模式**：后续 Step 7 按 `disclosure_builder.md` 的 `[软件模式]` 章节结构执行（mermaid 系统框图 + 流程图 + 符号公式）
- **B → 结构模式**：后续 Step 7 按 `disclosure_builder.md` 的 `[结构模式]` 章节结构执行（结构附图说明 + 部件编号 + 状态描述 + 结构权利要求）；**须** `Read` `references/structural_patent_requirements.md`
- **C → 混合模式**：同时参考两种模式的章节指导，以实物结构为主线、方法/控制策略为从属
- **D → 暂缓**：扫描材料后在 Step 3-4 再判定，默认按软件模式起步

### Q2：专利类型倾向

```
是否已有倾向的权利要求类型？方法 / 系统 / 装置 / 暂不确定
```

**[结构模式补充提示]**：结构类专利通常以「装置」独权为主（结构特征限定），可配合「方法」从属权利要求。若用户选「方法」但 Q1.5 为 B 类，应提醒结构类专利优先用结构特征限定独立权利要求。

### Q3：文头联系人

```
交底书文头的技术联系人是否需要占位？需要则提供姓名/电话/邮箱；不需要则全部写「待填写」。
```

### Q4（仅结构模式）：附图现状

```
是否已有附图或草图？
A. 有 CAD/Visio 正式附图
B. 有手绘草图或 AI 生成参考图（需后续整理为正式附图）
C. 尚无附图，需要在交底书中描述附图需求供制图人员参考
```

## 汇总

将已确认项用 3–6 行 bullet 复述给用户（含已确认的专利技术领域/模式），再进入 `project_scan.md` 对应的扫描阶段。**结构模式**时提醒用户：交底书将采用结构附图+部件编号体系，正式附图需为黑白线稿。

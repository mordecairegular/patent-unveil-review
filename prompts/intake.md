# 边界与输入录入（Step 1）

## 用途

在开始专利挖掘与交底书撰写前，用少量问题收敛案件边界。信息不全时可跳过，由 Agent 后续推断并注明假设。本技能主线聚焦工程方法、软件系统、数据处理、控制测算、工程仿真/测算工具类中国发明专利。

## 可选开场

```text
为便于判断是否适合本技能，并组织专利点与交底书，我先确认几项边界；可跳过，将按已有材料推断。
```

## 问题序列

### Q1：技术主题或产品模块

```text
请用一句话描述本方案所属技术主题或产品模块，例如「源荷储逐时匹配测算」「风机周边敏感建筑识别」「专利交底书技术闭环校核」「工业质检缺陷分级」。
```

### Q1.5：技术方案分流

**A 路线（主模式）**：核心创新点属于以下任一类时，进入标准流程 Step 2-8。

- 工程计算/测算方法：如逐时能量平衡、容量配置枚举、匹配度评估、工程参数校核。
- 软件系统/数据处理流程：如自动识别、特征提取、结构化解析、一致性校核、质量门禁。
- 控制策略/调度方法：如功率分配、SOC 约束、充放电策略、设备运行策略。
- 算法与工程场景结合：如基于遥感影像的目标识别、基于运行数据的异常诊断。

**B 路线（结构有限辅助）**：核心创新点为机械结构、零部件构造、连接/密封/装配关系等时，Agent 须立即告知以下限制，再由用户决定是否继续：

1. 本技能不对结构类方案做专利点挖掘和创造性评估。
2. 需用户自备 CAD/Visio 线稿、工程图、标注草图或等价附图来源。
3. 需用户提供明确物理机理依据，包括实验、仿真、标准、行业数据或工程验证说明。
4. 用户确认继续时，本技能仅辅助文字撰写、附图标记一致性、结构权利要求格式指导，并引用 `references/structural_patent_requirements.md`。
5. 自检中不输出 PASS，最高输出 WARN，并注明「结构类有限辅助，创造性评估需代理人或发明人自行完成」。

**C 路线（不适合本技能）**：纯商业规则、管理流程、经营方法、信息展示、学术发现、无技术手段的评分/推荐/排序等，Agent 应直接说明不适用本技能，并建议用户先确认是否存在可专利的技术手段、技术问题和技术效果。

### Q1.6：方案成熟度预检（Agent 内部评估）

Agent 默认从用户材料中判断以下四项，不逐条追问，除非缺口会阻断后续流程：

- 是否有可运行代码、工具原型或可复现流程。
- 是否有真实工程数据或明确输入数据格式。
- 是否有可量化技术效果指标，如准确率、耗时、弃电率、匹配度、错误率、稳定性等。
- 技术流程能否拆解为可描述的步骤和模块。

四项中满足 2 项及以上，视为「方案成熟度充分」，正常推进。满足 0-1 项时，在 Step 3-4 授权可行性初筛中标注「方案成熟度不足，建议先完善原型或补充数据再启动正式专利流程」。

### Q2：权利要求类型倾向

```text
是否已有倾向的保护组合？默认按「方法 + 系统/装置 + 电子设备 + 计算机可读存储介质」组织；也可指定为方法 / 系统 / 装置 / 暂不确定。
```

若用户技术方案同时包含结构改进和方法/控制改进，默认建议将方法/控制侧作为主线保护；结构侧视材料支撑情况写入从属权利要求、实施例，或另行考虑实用新型/结构类申请。

### Q3：文头联系人

```text
交底书文头的技术联系人是否需要占位？需要则提供姓名/电话/邮箱；不需要则全部写「待填写」。
```

### Q4（仅 B 路线）：附图与机理现状

```text
结构有限辅助需要确认：是否已有正式附图或草图？是否已有实验/仿真/标准/工程验证依据？
A. 有正式附图 + 机理/验证依据
B. 有草图或参考图，但机理/验证依据待补
C. 尚无附图或机理依据
```

若选择 B 或 C，Agent 仍可整理待补材料清单，但不得输出 PASS 或“授权可行”结论。

## 汇总

将已确认项用 3-6 行 bullet 复述给用户，至少包括技术主题、路线判断、方案成熟度判断、默认保护组合和关键待补材料。A 路线进入 `project_scan.md`；B 路线仅在用户确认理解限制后继续，并读取 `references/structural_patent_requirements.md`；C 路线停止专利交底书流程，改为说明不适用原因和可补强方向。
